Technical foundation

What runs technically: Four pillars of continuous monitoring

Network IDS/IPS

Network-based Intrusion Detection & Prevention (IDS/IPS)

Enginsight Core monitors your entire network traffic in real time – internally between systems and externally to the internet.

What is detected
Port scans and reconnaissance
Brute-force attacks across SSH, RDP and VPN
Lateral movement between systems
Command & control communication and data exfiltration attempts
Example: A laptop starts scanning Port 445 (SMB) on all servers at 11:47 PM. Core detects the pattern, classifies it as reconnaissance and alerts immediately.

File Integrity Monitoring

File Integrity Monitoring (FIM)

Protects critical system files, configurations and application binaries from unauthorized changes.

What is monitored
Operating system files like /etc/passwd and registry entries
SSH and RDP configurations
Webserver and database configurations
Certificates, keys and critical application binaries
Example: An attacker creates a new user. FIM detects the change to /etc/passwd in real time and alerts – before the account is even used.

Monitors availability and performance. Detects anomalies indicating compromise.

Network traffic volume and process lists
Unexpected processes, logins and user activity

FAQ

Frequently asked questions about Enginsight Core

Do I need Core if I already have endpoint protection?

Yes. EDR protects laptops and workstations. Core monitors server infrastructure and network traffic. Both complement each other.

How long does installation take?

Plan for 1-2 days installation, 2-4 weeks baseline learning, and productive monitoring from day 1.

How many false positives?

During the first two weeks expect roughly 20-30%. From week 4 after tuning, this usually falls below 5%.

On-premise possible?

Yes. Cloud, hybrid or fully on-premise deployments are available.

What happens during a critical attack?

Without MDR you get the alert in the dashboard and via email so your team responds. With MDR the SOC assesses within 15 minutes and can isolate systems if needed.

Difference to pentest?

Pentest means a quarterly snapshot. Core means continuous 24/7 monitoring and detection of active attacks.
